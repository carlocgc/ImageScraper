# Project Maintenance Roadmap

## Goal

Finish moving planning out of repo-wide checklists and into smaller docs plus actionable GitHub issues.

## Next housekeeping steps

- close or update stale issues whose work already landed
- create or refresh issues for the open work tracked in `docs/`
- decide which older ideas should be marked wont-fix instead of lingering
- align milestones with the current docs split:
  - refactor / cleanup
  - UI / product
  - runtime / distribution
  - provider expansion

## Recommended issue pass

Start with the issues that are most obviously stale:

- close already-done work such as the big UI modularisation and preview-panel items
- update surviving issues for things that are still real, such as explicit Reddit deauthorisation and provider additions
- decide explicitly on low-signal older ideas like PCH work, a file logger, Steam support, or other backlog noise instead of carrying them forever

## Source-of-truth rule

- keep repo docs at the workstream level
- keep GitHub issues at the task level
- prune docs when they stop being useful, rather than letting them become another monolithic plan file
